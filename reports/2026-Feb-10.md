**깨달은 점**
1. 내 컴에서 하고 git push 하고 서버에서 다시 clone 한다. 전용 init_server_git.sh 필요
2. render라는 온라인 24시간 서버 서비스가 있었고, 아주 개꿀이다.
3. 내 서버에서도 쓰고, 여기서도 써도 될 것 같다.




**개발 일지 — 오늘 진행 내용**

**1. 프록시 서버 구축 및 보안 구조 점검**

* Express 기반 Coinone 주문 프록시 서버 구성
* JWT 인증 미들웨어 적용 방식 검토
* n8n 자동화 요청을 위한 별도 시그널 API(`/api/signal`) 설계
* Access Key / Secret Key를 서버에서만 사용하도록 구조 유지

**2. 환경변수 및 보안 관리**

* `.env` 파일 사용 방식 정리
* GitHub에는 `.env` 및 `node_modules`를 제외하고 관리하도록 수정
* Render 환경 변수 설정 방식 확인 및 적용

**3. Render 배포 및 트러블슈팅**

* 초기 배포 시 `package.json` 누락 문제 해결
* Yarn / npm 혼용 문제로 발생한 의존성 충돌(fsevents) 분석
* Build Command를 `npm install`로 통일
* 서버 포트를 `process.env.PORT` 기반으로 수정 필요성 확인

**4. Docker 도입 검토**

* Docker 이미지와 컨테이너 개념 정리
* TypeScript 서버를 Docker로 빌드/실행하는 구조 이해
* 로컬 서버 상시 운영 vs 클라우드 운영 전략 비교

**5. 인프라 방향성 정리**

* Render = 외부 접근 가능한 Public Endpoint 역할
* 개인 서버 = 실제 자동매매/AI 처리 엔진 역할
* 두 환경을 혼합 사용하는 구조의 장점 파악

**6. 현재 상태**

* Render에서 서버 실행 로그 확인 (포트 감지 및 재시작 정상)
* Docker Desktop 정상 실행 상태 확인

**7. 다음 진행 예정**

* Render URL을 이용한 외부 API 호출 테스트
* n8n → 프록시 → Coinone 주문 흐름 실검증
* Docker 기반 장기 운영 환경 구축 여부 결정
